## 1. 全体構成

### 4画面遷移

```
[待機画面] ─タッチ/センサー─> [検索画面] ─検索実行─> [検索結果画面] ─カード選択─> [詳細画面]
    ↑                                                                              │
    └────────── 60秒無操作で自動復帰 ─────────────────────────────────────────────┘
```

### 画面遷移マトリクス

| From / To | 待機 | 検索 | 結果 | 詳細 |
|---|---|---|---|---|
| **待機** | — | タッチ/センサー | — | — |
| **検索** | 60秒無操作・戻るボタン | — | 検索実行 | — |
| **結果** | 60秒無操作 | 戻るボタン | — | カード選択 |
| **詳細** | 60秒無操作 | — | 戻るボタン・関連事例 | — |

### 既存検索UIとの関係

- 既存OSMIL番号検索・カテゴリ検索を **温存**（AI対話は追加レイヤー）
- AI応答失敗時のフォールバック導線として常時表示

## 3. 待機画面

### コンセプト

シーマの導入事例写真が円形にトリミングされ、白いキャンバス上を有機的に漂い続ける。
「キャンバスになろうよ。」の世界観を、空間に並ぶ事例群として可視化。

### アニメーション仕様

| 項目 | 確定値 |
|---|---|
| 形状 | 円形（写真を丸くトリミング、`CornerRadius = diameter/2`） |
| 同時表示数 | 20〜30個 |
| サイズ範囲 | 直径 50〜250px（大小混在） |
| 漂い | 位置を独立サイン波で揺らす（DriftFrequencyX/Y, DriftAmplitudeX/Y） |
| 拡縮 | サイズも独立サイン波で 0.7〜1.3倍に変動（PulseFrequency） |
| フェードイン | 1.5秒 |
| 表示維持 | 2〜4秒（個別ランダム） |
| フェードアウト | 1.5秒 |
| Z-Index | ランダム（重なりあり） |
| 写真なし時の代替 | シーマブルー系グラデーション（`#0066CC` `#3388DD` `#6AAAE8` `#99CCFF`） |

### ロゴ

- 位置：画面上部中央
- 表示：「CIMA」テキスト（仮、後で画像差替）
- フォントサイズ：48px Bold
- 色：シーマブルー

### 下部テキスト

- 位置：画面下部中央、下部から80px上
- フォントサイズ：28px
- 色：`#666666`
- 表示：「🎙️ 話しかけてください」と「👆 タッチしてください」を **3秒ごとにフェード切替**

### タッチ・センサー反応

- タッチパネル接触 または 人感センサー検知 → 検索画面へ遷移
- **TBD：遷移先（音声入力直行 vs ホームメニュー経由）**

## 4. 検索画面

### 構成

```
┌─────────────────────────────────────┐
│ [← 戻る]              [CIMA]            │
│ ──────────────────────────────────── │
│                                          │
│  🎙️ 音声で検索                          │
│  ┌────────────────────────────────┐    │
│  │ 「ガーナーホテルの事例を…」     │    │
│  │  〜〜波形アニメーション〜〜      │    │
│  └────────────────────────────────┘    │
│                                          │
│  ── または ──                            │
│                                          │
│  ⌨️  キーワード入力                      │
│  ┌────────────────────────────────┐    │
│  │                                 │    │
│  └────────────────────────────────┘    │
│                                          │
│  よく使われる検索：                       │
│  [ホテル][商業施設][イベント]            │
│  [LEDビジョン][屋外][屋内]               │
└─────────────────────────────────────┘
```

### 音声入力UI

- マイクボタン：プッシュ・ツー・トーク方式（タップ開始、再タップ停止）
- 録音中表示：「お話しください」+ 波形アニメーション
- 自動終了：最大15秒 or 無音3秒
- 認識結果プレビュー：[送信] [やり直す]（誤認識修正の機会を提供）

### キーボード入力UI

- 多言語IME対応（日本語・英語・中国語・韓国語）
- サジェスト・候補表示
- よく使われる検索タグ：[ホテル][商業施設][イベント][LEDビジョン][屋外][屋内]

### 既存検索UI（温存）

- 「展示番号で検索」「ゾーン一覧から探す」を併存表示
- AI対話の障害時の自動フォールバック導線

## 8. インタラクション仕様

### タッチ操作

| 操作 | 動作 |
|---|---|
| タップ | 選択・遷移・ボタン押下 |
| 左右スワイプ | 写真切替、画面遷移 |
| ピンチイン/アウト | 写真拡大縮小 |
| 長押し | （現時点未定義） |

### 音声操作

- マイクボタン押下 → 録音開始
- 再押下 or 無音3秒 → 録音終了 → 認識結果プレビュー
- NAVIGATION インテント対応：「次の事例」「戻る」「ホームへ」

### 自動復帰

- 60秒無操作 → フェードアウト警告10秒（「まもなく待機画面に戻ります」「対話を続ける」ボタン）
- 警告中タップで継続可
- さらに10秒無操作で待機画面復帰

### フォールバック

- AI応答10秒超過 or エラー → 既存検索UI（番号検索・カテゴリ検索）へ自動切替＋案内表示
- マイク無反応 → タッチ入力モードに自動切替

## 9. アクセシビリティ

- **音声/タッチ併用**：どちらか片方が使えない来場者にも対応
- **フォントサイズ**：本文28px以上推奨（ショールーム距離想定）
- **色覚配慮**：青系単一に依存しない（テキストラベル併記）
- **多言語**：日・英・中（簡）・韓を基本（実装フェーズで詳細）
- **音声出力（TTS）**：オプションで提供想定（VOICEVOX等、未確定）
