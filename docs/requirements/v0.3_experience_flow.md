## 1. 体験フロー全体像

```
来場者 ──音声発話──▶ ASR（音声認識）
                        │
                        ▼
                  インテント分類（F-11）
                        │
          ┌─────────────┼──────────────────┐
          ▼             ▼                  ▼
     DIRECT_RECALL  FILTER_SEARCH     CONSULTATION
     SPEC_SEARCH    TERM_EXPLAIN      COMPARISON
                    NAVIGATION
          │             │                  │
          ▼             ▼                  ▼
     事例直接表示    絞り込み結果表示   対話応答＋事例案内
          │             │                  │

                        │
                  関連事例自動提示（F-16）
                        │
                  音声サマリー読み上げ（F-15）
```

## 2. シナリオA：用途指定の音声検索

**インテント分類：FILTER_SEARCH**

### 来訪者発話

> 「ホテルのロビーに入れるLEDビジョンの事例を探してほしいんですが」

### AI解釈フロー

```json
{
  "intent": "FILTER_SEARCH",
  "confidence": 0.91,
  "extracted_params": {
    "client_type": "hotel",
    "location_type": "lobby",
    "category_secondary": "led_vision",
    "indoor_outdoor": "indoor"
  }
}
```

### 画面表示内容

1. **検索中アニメーション**（〜0.5秒）
   - 「ホテルロビーのLEDビジョン事例を検索しています...」

2. **検索結果一覧**（3〜5件）

   | 順位 | 施主名 | 概要 | ピッチ |
   |---|---|---|---|
   | 1 | ○○ホテル | ロビー壁面 P1.9 フルカラー | 1.9mm |
   | 2 | △△リゾート | エントランスLEDビジョン | 2.5mm |
   | 3 | □□グランドホテル | フロントカウンター上部設置 | 1.5mm |

3. **AI音声応答**
   > 「ホテルロビーへのLEDビジョン設置事例が3件見つかりました。
   > 一番上の○○ホテル様の事例では、壁面にP1.9のフルカラービジョンを
   > 設置しております。詳しく見る場合はタッチまたは『最初の事例を見せて』
   > とお声がけください。📍出典：CIMA-XXXXX『○○ホテル』」

### 来訪者の次アクション

- タッチまたは「最初の事例を見せて」→ シナリオA-2（詳細表示）へ
- 「もっと細かく絞って」→ シナリオC（仕様指定）へ
- 「別の用途も見たい」→ 新規検索

### シナリオA-2：詳細選択後

**来訪者発話**

> 「一番上のを見せて」

**画面表示**

- 事例詳細ページ全画面表示
- OSMIL経由で大型ディスプレイへ映像信号切替
- 画面下部に関連事例3件のサムネイル

## 3. シナリオB：固有名指定の直接呼び出し

**インテント分類：DIRECT_RECALL**

### 来訪者発話

> 「カワスイの事例を見せてください」

### AI解釈フロー

```json
{
  "intent": "DIRECT_RECALL",
  "confidence": 0.97,
  "extracted_params": {
    "target_name": "カワスイ",
    "matched_id": "CIMA-18433"
  }
}
```

### 画面表示内容

1. **マッチング処理**（〜0.3秒）
   - `keywords_for_voice` フィールドから「カワスイ」を完全一致検索

2. **事例詳細ページへ直行**
   - 一覧を経由せず、CIMA-18433 の詳細ページへ即遷移
   - OSMIL経由で大型ディスプレイへ映像信号切替

3. **AI音声応答**
   > 「川崎水族館（カワスイ）様の事例をお呼び出しします。
   > 屋内型LEDビジョンをエントランスに設置した事例です。
   > 📍出典：CIMA-18433『川崎水族館（カワスイ）』」

### 複数ヒット時の処理

同名が複数存在する場合は候補一覧を表示し、「どちらの○○ですか？」と音声確認する。

### ゼロヒット時の処理

> 「申し訳ございません、『△△』に該当する事例が見つかりませんでした。
> 似た用途や業種でお探しする場合はお知らせください。」

## 4. シナリオC：仕様指定の絞り込み

**インテント分類：SPEC_SEARCH**

### 来訪者発話

> 「1.9ミリピッチの屋内の事例を教えてください」

### AI解釈フロー

```json
{
  "intent": "SPEC_SEARCH",
  "confidence": 0.94,
  "extracted_params": {
    "pitch_mm": 1.9,
    "indoor_outdoor": "indoor"
  }
}
```

### 画面表示内容

1. **絞り込み処理**（〜0.4秒）
   - SQLiteで `pitch_mm = 1.9` かつ `indoor_outdoor = indoor` を検索
   - 該当件数をリアルタイム表示

2. **検索結果一覧**（最大10件）

   フィルタパネルにも「P1.9」「屋内」が自動反映される。

   | 順位 | 施主名 | 施設種別 | 設置場所 |
   |---|---|---|---|
   | 1 | ○○水族館 | aquarium | エントランス |
   | 2 | △△ホテル | hotel | ロビー |
   | 3 | □□商業施設 | commercial | アトリウム |

3. **AI音声応答**
   > 「P1.9ミリ（ピッチ1.9ミリ）の屋内事例が○件見つかりました。
   > P1.9は画素間隔が1.9ミリの高精細タイプで、近距離での視認性に優れております。
   > 📍出典：検索結果一覧」

## 5. シナリオD：抽象的相談（対話モードへの切替）

**インテント分類：CONSULTATION**

### 来訪者発話

> 「うちは商業施設のエントランスなんですが、何が適しているか教えてもらえますか」

### AI解釈フロー

```json
{
  "intent": "CONSULTATION",
  "confidence": 0.88,
  "extracted_params": {
    "client_type": "commercial",
    "location_type": "entrance"
  }
}
```

### 対話フロー

**ターン1：AI質問**
> 「商業施設のエントランスへの導入をお考えでしょうか。
> 少し詳しくお聞かせください。視聴距離はどのくらいを想定されていますか？
> たとえば、近くで見る場合はP1.9〜P2.5、遠くから見る場合はP4以上が
> 一般的な目安になります。」

**来訪者発話（ターン2）**
> 「3〜5メートルくらいから見ることが多いと思います」

**AI応答（ターン2）**
> 「視聴距離3〜5メートルの場合は、P2.5〜P3.0程度が適しています。
> 商業施設エントランスへのP2.5設置事例が2件ございます。
> ご覧になりますか？
> 📍出典：CIMA-XXXXX『△△モール』、CIMA-YYYYY『□□ショッピングセンター』」

**来訪者発話（ターン3）**
> 「見てみたいです」

**AI応答（ターン3）→ 事例一覧へ遷移**

> 「では該当事例をお見せします。」

画面：事例一覧（P2.5 × 商業施設 × エントランス）へ自動遷移。

### ターン上限到達時

5ターンを超えた場合：
> 「詳細な要件については、担当営業よりご説明いたします。
> 引き続き事例を自由にご覧ください。」

## 6. シナリオE：関連事例への横展開

**トリガー：事例詳細表示中に関連事例タッチ、または音声指示**

### 前提

来訪者は「カワスイ（川崎水族館）」の事例詳細を閲覧中。

### 来訪者発話（パターン1：用途ベース）

> 「同じような水族館の事例は他にありますか」

**AI処理**

- 現在表示事例の `client_type = aquarium` で FILTER_SEARCH を実行
- 現在事例を除外してランキング

**AI音声応答**
> 「水族館への導入事例がほかに○件ございます。ご覧ください。
> 📍出典：検索結果一覧」

### 来訪者発話（パターン2：製品タイプベース）

> 「COBタイプの事例を他にも見せてほしいんですが」

**AI処理**

- `use_case_tags` に「COB」を含む事例をベクトル検索
- 現在事例を除外

**AI音声応答**
> 「COB（Chip On Board）タイプの事例がほかに△件ございます。
> COBは画素が基板に直接実装された方式で、耐水性・耐久性に優れております。
> 📍出典：検索結果一覧」

### 画面下部の関連事例パネル

事例詳細ページ表示中、常に画面下部に関連事例3件を表示する。

| サムネイル | 施主名 | 関連理由 |
|---|---|---|
| 🖼 | △△水族館 | 同じ用途（水族館） |
| 🖼 | □□ホテル | 同じ製品タイプ（COB） |
| 🖼 | ○○商業施設 | 同じ設置場所（エントランス） |

## 7. 共通UI挙動

### 7.1 音声入力受付状態

| 状態 | 表示 |
|---|---|
| 待機中 | マイクアイコン点灯（白） |
| 発話検出中 | マイクアイコン点灯（青）＋波形アニメーション |
| 処理中 | マイクアイコン点灯（オレンジ）＋スピナー |
| 応答表示中 | マイクアイコン点灯（緑）＋割り込み受付中 |

### 7.2 割り込み発話

応答表示・読み上げ中でも新規発話を検出した場合、現在の処理を即停止して
新規クエリの処理を開始する。

### 7.3 無操作タイムアウト

3分間の無操作でトップ画面（カテゴリブラウズ）へ自動リセット。
リセット30秒前に「引き続き事例をご覧になりますか？」と音声通知。
