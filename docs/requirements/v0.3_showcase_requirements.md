## 1. プロジェクト概要

### 1.1 目的

シーマショールーム来訪者に対し、音声入力で関連導入事例を即座に呼び出し、
大型ディスプレイで提示する営業デモツール「Argus Canvas Showcase」を開発する。

### 1.2 背景

シーマは映像機器・LEDディスプレイ・音響機器のシステム設計・施工・レンタルを
手がけており、多数の導入実績を保有している。しかし来訪者が事例を探す際、
キーワード検索では意図に合致した事例を素早く見つけることが難しい。

音声入力による自然言語検索と、ローカルLLMによる意図解釈を組み合わせることで、
営業担当者の説明負担を軽減しつつ、来訪者の自走型体験を実現する。

### 1.3 スコープ

| 対象 | 内容 |
|---|---|
| 開発主体 | 株式会社アーガス企画 |
| 納品先 | シーマ（グループ会社） |
| 稼働場所 | シーマショールーム内 大型ディスプレイ端末 |
| 利用者 | ショールーム来訪者（営業同席あり） |
| AIエンジン | Gemma 4 E4B（ローカル動作） |

### 1.4 スコープ外

- 受注管理・顧客情報管理機能
- 外部ネットワーク連携
- スマートフォン・タブレット対応
- 多言語対応（日本語のみ）

## 2. 機能要件

### カテゴリA：音声駆動コンテンツ呼び出し（最重要）

#### F-11：音声インテント解釈

| 項目 | 内容 |
|---|---|
| 概要 | 音声入力テキストを7種インテントに分類し、後続処理に振り分ける |
| 分類器 | Gemma 4 E4B（Ollama経由） |
| 応答形式 | JSON（intent / confidence / extracted_params） |
| 閾値 | confidence < 0.5 の場合は CONSULTATION として扱う |

**7インテント一覧**

| インテント | 説明 | 代表発話例 |
|---|---|---|
| DIRECT_RECALL | 固有名称を指定した直接呼び出し | 「カワスイの事例を見せて」 |
| FILTER_SEARCH | 用途・業種・環境による絞り込み | 「ホテルロビーに合うLEDを探して」 |
| SPEC_SEARCH | 仕様値による絞り込み | 「1.9ミリピッチの屋内事例」 |
| CONSULTATION | 抽象的な相談・提案依頼 | 「商業施設のエントランスに何が適しているか」 |
| TERM_EXPLAIN | 用語・製品の解説要求 | 「COBタイプって何ですか」 |
| COMPARISON | 複数製品・事例の比較要求 | 「1.9ミリと2.5ミリの違いは」 |
| NAVIGATION | 画面操作指示 | 「次の事例を見せて」「戻って」 |

#### F-12：事例直接表示

| 項目 | 内容 |
|---|---|
| 概要 | DIRECT_RECALLインテント時、固有名に対応する事例ページへ直遷移 |
| マッチング | SQLiteの `keywords_for_voice` フィールドとの完全一致・部分一致 |
| 複数ヒット時 | 最上位スコアの事例を表示、他候補を下部に提示 |
| ゼロヒット時 | 「申し訳ございません、該当する事例が見つかりませんでした」と音声・テキストで通知 |

#### F-13：絞り込み検索

| 項目 | 内容 |
|---|---|
| 概要 | FILTER_SEARCH / SPEC_SEARCH インテント時、複数条件でフィルタリング |
| 絞り込み軸 | 用途タグ・LEDピッチ・屋内外・施設種別・設置年 |
| 結果件数 | 最大10件表示（スコア降順） |
| 表示形式 | サムネイル＋タイトル＋施主名＋要約（100文字） |

#### F-14：検索結果ランキング

| 項目 | 内容 |
|---|---|
| 概要 | ChromaDBのベクトル類似度とメタデータフィルタを組み合わせてスコアリング |
| ベクトル類似度重み | 0.7 |
| メタデータ一致重み | 0.3 |
| 足切り閾値 | コサイン類似度 0.7 未満は除外 |

#### F-15：音声サマリー読み上げ

| 項目 | 内容 |
|---|---|
| 概要 | 事例表示時、`summary_short` を音声合成で読み上げる |
| TTS | Windows Speech API（標準）/ VOICEVOX（オプション） |
| スキップ | 来訪者の割り込み発話で即停止 |
| 読み上げ対象 | 検索結果の上位3件の概要、または選択事例の詳細サマリー |

#### F-16：関連事例自動提示

| 項目 | 内容 |
|---|---|
| 概要 | 事例詳細表示時、関連事例3件を画面下部に提示 |
| 関連度算出 | `related_cases` フィールド優先 → ベクトル類似度補完 |
| ラベル表示 | 「同じ用途」「同じ製品タイプ」など関連理由を表示 |

#### F-17：階層ナビゲーション

| 項目 | 内容 |
|---|---|
| 概要 | NAVIGATION インテントで「次・前・戻る・一覧に戻る」を音声操作 |
| 対応発話 | 「次を見せて」「前に戻って」「一覧に戻って」「最初から」 |
| アニメーション | スライドトランジション（200ms） |

---

### カテゴリB：既存検索機能との共存

#### F-18：カテゴリブラウズ（温存）

既存のカテゴリ階層ブラウズUIを維持する。タッチ操作で施設種別・製品カテゴリを
ドリルダウンできる既存体験を損なわない。AIレイヤーは追加機能として上乗せする。

#### F-19：キーワード検索

既存のキーワード検索フォームを維持する。フォーム入力とF-11音声入力は
同じ検索パイプラインを共有し、一貫した結果を返す。

#### F-20：フィルタ機能

既存のフィルタパネル（施設種別・製品種別・屋内外・年代）を維持する。
音声検索で抽出したパラメータは、このフィルタパネルにも反映して視覚的に確認できるようにする。

---

### カテゴリC：対話型応答

#### F-21：用途相談対話

| 項目 | 内容 |
|---|---|
| 概要 | CONSULTATION インテント時、要件ヒアリング→事例推薦の対話を実施 |
| ターン数 | 最大5ターン（それ以上は「営業担当にお繋ぎします」） |
| 推薦根拠 | 必ず事例ID・施主名を明示 |

#### F-22：製品仕様説明

TERM_EXPLAIN インテント時、事例データ中の製品仕様フィールドと
`summary_long` を根拠に説明する。RAGコンテキスト外の仕様は回答しない。

#### F-23：比較応答

COMPARISON インテント時、対象となる事例・製品を横並びで比較表示する。
比較軸は「ピッチ・輝度・用途・設置環境・事例年」を標準とし、
来訪者の発話内容に応じて軸を絞り込む。

#### F-24：出典明示

すべての生成応答末尾に「📍出典：事例ID『施主名』」を表示する。
複数出典がある場合はカンマ区切りで列挙する。出典なし応答は警告マークを付与する。

## 3. 非機能要件

### 3.1 パフォーマンス

| 指標 | 目標値 | 計測方法 |
|---|---|---|
| 初回トークン表示 | 1.5秒以内 | 発話終端〜最初のトークン描画 |
| 生成速度 | 40 tokens/sec以上 | ストリーミング計測 |
| 検索レイテンシ | 200ms以内（ChromaDB） | APIレスポンス計測 |
| 起動時間 | 30秒以内 | アプリ起動〜操作可能状態 |

### 3.2 可用性・運用

| 項目 | 内容 |
|---|---|
| 連続稼働 | 24時間連続稼働（ショールーム営業時間内） |
| 更新方式 | USBメモリ差込による30秒更新（CMSレス） |
| ネットワーク | 完全オフライン動作（インターネット不要） |
| 自動復帰 | 異常終了時は自動再起動（Windows サービス化） |

### 3.3 プライバシー・セキュリティ

| 項目 | 内容 |
|---|---|
| 音声データ | ローカル処理のみ、外部送信なし、録音非保持 |
| 対話履歴 | セッション終了時に全削除（個人特定情報は一切保持しない） |
| 事例データ | 端末内SQLite・ChromaDB のみ、クラウド連携なし |

### 3.4 ハードウェア要件（推奨）

| 項目 | 推奨スペック |
|---|---|
| CPU | Intel Core i7-13700 以上 |
| GPU | NVIDIA RTX 4070 以上（VRAM 12GB以上） |
| RAM | 32GB以上 |
| ストレージ | SSD 512GB以上 |
| CUDA | 12.x または 13.1（13.2は使用禁止） |
| OS | Windows 11 Pro |

## 4. システム構成

```
┌─────────────────────────────────────────────────────────┐
│                    WPF / .NET 8（UI層）                  │
│  音声入力 → ASR → テキスト → 表示制御 → 画面描画        │
└────────────────────────┬────────────────────────────────┘
                         │ HTTP (REST)
┌────────────────────────▼────────────────────────────────┐
│              Showcase.Core（ビジネスロジック）            │
│  IntentClassifier → SearchOrchestrator → ResponseGen    │
└──────┬─────────────────┬──────────────────┬─────────────┘
       │                 │                  │
┌──────▼──────┐  ┌───────▼───────┐  ┌──────▼──────────────┐
│   Ollama    │  │   ChromaDB    │  │       SQLite        │
│  Gemma 4   │  │ （ベクトルDB） │  │   （メタデータ）    │
│    E4B      │  │               │  │                     │
└─────────────┘  └───────────────┘  └─────────────────────┘
```

### 4.1 レイヤー構成

| レイヤー | プロジェクト | 役割 |
|---|---|---|
| UI層 | `Showcase.UI` | WPF画面・音声入出力・表示制御 |
| ビジネスロジック | `Showcase.Core` | インテント分類・検索・応答生成オーケストレーション |
| AI連携 | `Showcase.AI` | Ollamaクライアント・プロンプト管理 |
| データアクセス | `Showcase.Data` | ChromaDB・SQLiteアクセス |
| バッチ処理 | `scripts/` | 事例データ収集・ベクトル化・インポート（Python） |

### 4.2 OSMIL連携

UI層はアーガス企画独自の映像表示制御エンジン「OSMIL」と協調動作する。
事例詳細ページ表示時、OSMIL APIへコンテンツIDを通知し、
大型ディスプレイへの全画面映像切り替えを実現する。
